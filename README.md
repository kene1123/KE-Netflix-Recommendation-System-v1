# 🎬 Ke-Netflix — Production-Style Movie Recommendation System

A full-stack hybrid recommendation engine built end to end: data pipeline, behavioral simulation, dual ML algorithms, automated scheduling, and a live interactive Streamlit app — all running on real infrastructure (PostgreSQL on Neon, GitHub Actions, Streamlit Cloud).

This is not a notebook demo. It's a production-structured system with idempotent pipelines, scheduled automation, explainable recommendations, and a deployed front end.

## What it does

Ke-Netflix simulates a personalized streaming experience. Each user gets recommendations generated by three independent algorithms — content-based filtering, collaborative filtering (SVD), and a hybrid blend of both — each with a human-readable explanation of *why* that movie was recommended.

| | |
|---|---|
| 🎞 **Movies** | 9,800+ titles, cleaned and enriched with TMDB posters |
| 👥 **Users** | 610 real MovieLens users with simulated ongoing behavior |
| ⭐ **Activity** | 109,000+ ratings and watch events |
| 🎯 **Recommendations** | 18,300+ across content / collaborative / hybrid |
| 🤖 **Model** | SVD (50 latent factors) — RMSE 0.99, Precision@10 ~15% |

## Architecture

```
Data Source Layer
  MovieLens dataset + TMDB API enrichment
        │
Ingestion & Cleaning Layer
  pipelines/clean_and_sync.py
  └─ Unicode normalization, article reordering, year extraction
  └─ TMDB poster fetch + new-movie discovery
        │
Database Layer (PostgreSQL / Neon)
  movies · users · user_activity · user_profiles
  recommendations · model_metrics · pipeline_runs
        │
Simulation Layer
  pipelines/simulation.py
  └─ Derives genre preference + rating personality per user
  └─ Backfills realistic watch_time from rating behavior
  └─ Generates new watch events weighted by taste profile
        │
ML Layer
  pipelines/features.py        → TF-IDF + precomputed similarity
  models/content_model.py      → content-based filtering
  models/collaborative_model.py → SVD matrix factorization
  pipelines/reccomend.py       → hybrid scoring (0.6 collab + 0.4 content)
  pipelines/evaluation.py      → RMSE / Precision@10
        │
Automation Layer
  pipelines/automate.py
  └─ Tracks last run via pipeline_runs table
  └─ Refreshes only users with new activity
  └─ Daily: collaborative + hybrid · Weekly: full recompute
  .github/workflows/clean_and_sync.yml   (weekly TMDB sync)
  .github/workflows/automate.yml         (daily/weekly refresh)
        │
Application Layer
  app/app.py — Streamlit interactive dashboard
  └─ Personalized home feed, explainable recommendations
  └─ Watch history, genre analytics, model insights
  └─ Live search with genre-similarity slicer
```

## Engineering principles

- **No hardcoded credentials** — all secrets via `.env` locally, GitHub Secrets / Streamlit Secrets in production
- **Idempotent pipelines** — every script is safe to re-run without duplicating data
- **Self-healing schema** — pipelines check and repair missing columns/constraints on every run
- **Explainability by design** — every recommendation stores its `reason` text alongside the score
- **Pooler-safe DB access** — all large queries use chunked `fetchmany()` pagination, compatible with PgBouncer transaction pooling
- **Separation of concerns** — pipelines write, the app only reads; no ML logic lives in the UI layer

## Running locally

```bash
# 1. Clone and install
git clone <your-repo-url>
cd ke-netflix-recommendation-system
pip install -r requirements.txt

# 2. Configure environment
cp .env.example .env
# fill in DB credentials + TMDB_API_KEY

# 3. Bootstrap the database (first time only)
python -m pipelines.clean_and_sync --bootstrap
python -m pipelines.simulation
python -m pipelines.features
python -m pipelines.reccomend
python -m pipelines.evaluation

# 4. Launch the app
streamlit run app/app.py
```
